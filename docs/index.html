Managing initiatives, surveys & reports for education

ASRS is a web application for creating initiatives, distributing and collecting surveys, generating per-initiative reports with filters and trend analysis, and sharing results through QR codes and downloadable exports.

01 — Initiatives

Create & manage initiatives

Initiatives are the central organizing unit of the system. Each initiative holds attributes, questions, and settings that drive survey distribution and report generation. Staff and admins create initiatives; public users interact with the surveys they produce.

02 — Reporting Engine

Reports & analytics

Define, generate, and snapshot reports per-initiative. Reports support filters, sorts, and trend configuration — with optional AI-powered insights. Long-running report generation is handled by background processing.

Filters & Sorts

Narrow report data by any initiative attribute. Sort results to surface the information that matters most for decision-making.

Trend Configuration

Configure trend analysis across time periods to identify patterns in survey responses and initiative performance over time.

AI Insights

Optional AI-powered analysis layer that generates summary insights from report data, highlighting key findings and anomalies.

Report Generation

Create reports tied to specific initiatives. Each report is a generated snapshot of the initiative's current data — saved so it can be viewed, compared, or exported later. Background jobs handle long-running generation without blocking the UI.

API → src/app/api/reports/route.js
UI → src/app/reporting/ (report pages)
Events → report.created
CSV PDF AI Insights

Snapshots & History

Reports are saved as snapshots — frozen views of initiative data at a specific point in time. This lets staff compare results across periods and track how an initiative's metrics evolve over its lifecycle.

CSV PDF

Exports & Artifact Bundles

Download reports and initiative data as CSV, PDF, or PNG assets. The export system supports both individual downloads and bundled artifact packages for sharing or archival. Background processing handles large export jobs asynchronously.

Formats: CSV · PDF · PNG · SVG  |  Delivery: direct download · bundled artifact · background job

03 — QR Codes

QR code distribution & tracking

Generate QR codes that link directly to surveys or reports. Download them as PNG or SVG for print or digital use, and track every scan and conversion to measure engagement.

Generation & Download

Create QR codes for any survey or report. Download as high-resolution PNG or vector SVG — ready for posters, handouts, slides, or digital distribution. Each QR code links directly to its target and is managed through the QR Code Manager UI.

API → src/app/api/qr-codes/*
UI → src/components/QRCodeManager.js
PNG SVG

Scan & Conversion Tracking

Every QR code scan is recorded. The system tracks total scans, unique visitors, and conversions (users who complete the target survey or view the target report). Domain events like qr.scanned are emitted for real-time processing.

04 — Data Model

Core entities

Four primary entities and their relationships.

Initiative Report QR Code User

An Initiative is a named project holding questions and settings used to collect or summarize data. A Report is a generated snapshot of initiative data — saved for viewing or export. A QR Code is a shareable link that redirects users to a survey or report and records scan activity. User roles (public, staff, admin) determine what actions a user can perform.

05 — Access Control

Roles & permissions

Session-based authentication with three role tiers controlling what each user can see and do.

Public

Public User

Take surveys distributed via QR codes or direct links. View public-facing reports. No initiative management or report generation access.

Staff

Staff

Create and manage initiatives. Generate and view reports. Create and distribute QR codes. Access survey response data and export reports.

Admin

Administrator

Full system access. Manage users, roles, and permissions. All staff capabilities plus administrative controls and configuration.

06 — Architecture

How it's built

A Next.js application with server-side API routes, a component-based frontend, and background processing for heavy tasks.

Next.js Pages API Routes React Components Database Background Jobs Domain Events

The UI lives in src/app/ and src/components/ — reporting pages, survey pages, QR manager, and sharing tools. Server endpoints under src/app/api/ handle initiatives, reports, and QR code operations. Shared logic and adapters live in src/lib/. Background processing handles long-running report generation and export jobs, emitting domain events (like report.created and qr.scanned) for real-time tracking.
